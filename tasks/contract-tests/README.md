# Контрактные тесты (pre-implementation, test-writer)

Файлы здесь — **независимые контрактные тесты**, написанные субагентом
`test-writer` на этапе постановки задачи (SENAR, правило 4): по сигнатурам и
критериям приёмки, **без доступа к реализации** (её ещё нет). Это структурная
защита от подгонки тестов под код.

Расширение `.test.ts.txt` намеренное: так файлы не подхватываются ни Vitest
(`include: src/**`), ни ESLint/Prettier, ни `svelte-check` — пока соответствующая
задача не взята в работу.

## Как использовать при старте задачи

1. Переименуй `<name>.test.ts.txt` → `<name>.test.ts` и перенеси в каталог рядом
   с будущей реализацией (путь указан в соответствующем Linear-issue).
2. Реализуй модуль под эти тесты. **Ассерты и ожидаемые значения не редактируй.**
   Падает — разбирайся в коде или в нечёткости критерия, а не правь тест.
3. Удали файл отсюда (он переехал в `src/`).
