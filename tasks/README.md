# Задачи

Основной трекер задач — **Linear** (проект mdpad++, команда MDP).

Файлы в этой папке — **зеркало для фиксации знаний**. У ИИ нет памяти
между сессиями. Без зафиксированных знаний одни и те же тупики
исследуются повторно.

## Когда создавать файл

Файл `tasks/<имя>.md` создаётся автоматически командой `/new-task <имя>`.
Его также можно создать вручную для задач, заведённых напрямую в Linear.

## Что хранить

При закрытии задачи (через `/done`) в файл добавляются:

- **Свидетельства верификации** — по каждому критерию
- **Тупики** — отвергнутые подходы (или "нет")
- **Решения** — неочевидные выборы (или "нет")
- **Известные проблемы** — внесённый техдолг (или "нет")

## Зачем

- Будущие ИИ-сессии видят историю и не переоткрывают тупики
- Метрики SENAR (FPSR, показатель тупиков) считаются по этим файлам
- Привязка к Linear: поле `Linear: MDP-XXX` в файле

## Что НЕ хранить здесь

- Идеи и черновики → Linear или `docs/`
- Документация фич → `docs/`
- Личные TODO → обычные заметки

В `tasks/` живут только задачи, прошедшие шлюз "Старт".
