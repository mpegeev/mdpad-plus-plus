# mdpad++ — Design Brief

> Этот документ — источник правды по визуальному языку mdpad++.
> Claude Code читает его при работе над любой UI-задачей.
> Обновляй его после любых изменений палитры, типографики или правил компонентов.

## Философия

mdpad++ — это **dev-tool**, а не consumer-приложение. Визуальный язык должен быть:

- **Тихим.** Интерфейс не соревнуется с контентом за внимание. Контент (текст заметок) — главный, UI — рамка.
- **Плотным.** Разработчики ценят плотность информации больше воздуха. Но без claustrophobia.
- **Предсказуемым.** Никаких неожиданных анимаций, всплывающих подсказок без повода, декоративных элементов.
- **Быстрым.** Анимации максимум 150мс, никаких "easing showcase". Ничего не должно мешать скорости работы.
- **Консистентным.** Один радиус, одна сетка отступов, один набор теней. Вариативность — признак непродуманности.

Ориентиры по духу: **Linear** (плотность и лаконичность), **VS Code** (информационная иерархия), **Zed** (скорость и ощущение нативности), **Typora** (тишина и фокус на контенте).

Анти-ориентиры: Notion (слишком много воздуха для dev-tool), Obsidian (визуально перегружен), любые "AI-first" редакторы с градиентами.

## Tech-решение

Используем **shadcn-svelte** (https://shadcn-svelte.com) как источник готовых компонентов. Это не библиотека в обычном смысле — компоненты **копируются в `src/lib/ui/`** и живут в репозитории как собственный код. Это даёт:

- полный контроль над стилями и поведением;
- отсутствие зависимости от внешней версионируемой библиотеки;
- Claude Code видит исходники и может их модифицировать под задачу.

Поверх shadcn-svelte работают **design tokens** — CSS custom properties, определённые в `src/styles/themes/`. Все компоненты используют только токены, никогда хардкод.

## Design tokens

### Цвета (темная тема по умолчанию)

Палитра построена на нейтральном тёмно-сером с одним акцентным синим. Никаких бренд-цветов сверх этого — минимум визуального шума.

```css
/* src/styles/themes/dark.css */
:root[data-theme="dark"] {
  /* Surfaces — от самого глубокого к самому поднятому */
  --bg-base: #1a1a1a; /* фон окна */
  --bg-elevated: #222222; /* сайдбар, таб-бар, статус-бар */
  --bg-overlay: #2a2a2a; /* попапы, dropdowns, tooltip */
  --bg-hover: #2f2f2f; /* hover состояние интерактивных элементов */
  --bg-active: #353535; /* нажатое / выбранное */
  --bg-selection: #264f78; /* выделение текста (VS Code стиль) */

  /* Text */
  --fg-primary: #e4e4e4; /* основной текст */
  --fg-secondary: #9a9a9a; /* подписи, подсказки, инактив */
  --fg-tertiary: #6b6b6b; /* декоративный текст, disabled */
  --fg-inverted: #1a1a1a; /* текст на ярком фоне (кнопка accent) */

  /* Borders — тонкие, почти незаметные */
  --border-subtle: #2e2e2e; /* между панелями */
  --border-default: #3a3a3a; /* у input, card */
  --border-strong: #4a4a4a; /* focus ring */

  /* Accent — единственный яркий цвет */
  --accent: #4f9cff; /* активная вкладка, кнопка, focus */
  --accent-hover: #67abff;
  --accent-muted: #2d4d7a; /* фон выделенной строки, badge */

  /* Semantic */
  --danger: #ff5c5c; /* несохранённые документы, ошибки */
  --warning: #e0c060; /* изменённые строки в gutter */
  --success: #6ec46e; /* добавленные строки в gutter, saved */

  /* Gutter / diff */
  --gutter-modified: var(--warning);
  --gutter-added: var(--success);
  --gutter-deleted: var(--danger);

  /* Syntax (для подсветки Markdown в raw-режиме) */
  --syntax-heading: #4f9cff;
  --syntax-emphasis: #e4e4e4;
  --syntax-strong: #ffffff;
  --syntax-code: #e0c060;
  --syntax-link: #67abff;
  --syntax-quote: #9a9a9a;
  --syntax-list: #9a9a9a;
  --syntax-hr: #3a3a3a;
}
```

### Светлая тема

Светлая тема — **не инверсия** тёмной, а самостоятельный набор. Проектируется в задаче MDP-27. Ключевые принципы: белый фон `#ffffff`, текст `#1a1a1a`, границы едва различимы, акцент — тот же синий `#4f9cff` для визуальной консистентности.

### Типографика

Две семьи шрифтов: моноширинный для редактора (raw режим, код) и system sans для UI-элементов и рендер-режима. Никаких подключаемых веб-шрифтов — только системные, чтобы приложение запускалось мгновенно.

```css
:root {
  /* Families */
  --font-ui:
    -apple-system, BlinkMacSystemFont, "Segoe UI", "Inter", Roboto,
    "Helvetica Neue", sans-serif;
  --font-mono:
    ui-monospace, "SF Mono", "Cascadia Mono", "JetBrains Mono", "Fira Code",
    Menlo, Consolas, monospace;
  --font-prose: var(--font-ui); /* для rendered markdown */

  /* Sizes (type scale 1.125 — "major second") */
  --fs-xs: 11px; /* бэйджи, мелкие подсказки */
  --fs-sm: 12px; /* sidebar, tabs, statusbar */
  --fs-base: 13px; /* body UI */
  --fs-md: 14px; /* editor default */
  --fs-lg: 16px; /* заголовки панелей */
  --fs-xl: 20px; /* rendered H2 */
  --fs-2xl: 24px; /* rendered H1 */

  /* Line heights */
  --lh-tight: 1.2;
  --lh-normal: 1.5;
  --lh-prose: 1.65; /* только для rendered markdown */

  /* Weights */
  --fw-regular: 400;
  --fw-medium: 500;
  --fw-bold: 600; /* никаких 700+ */
}
```

**Правило:** UI — `--fs-sm` или `--fs-base`. Текст в редакторе — `--fs-md`. Rendered markdown body — `--fs-md` с `--lh-prose`. Rendered H1/H2 — `--fs-2xl` / `--fs-xl`.

### Размеры и отступы

Сетка **4px**. Всё кратно четырём. Никаких "ну почти 7 пикселей".

```css
:root {
  --space-0: 0;
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 24px;
  --space-8: 32px;

  /* Радиусы — минимум, никаких больших скруглений */
  --radius-sm: 3px; /* input, button, tab */
  --radius-md: 4px; /* panel, card, dropdown */
  --radius-lg: 6px; /* overlay (только для больших поверхностей) */

  /* Высота интерактивных элементов */
  --h-control-sm: 24px; /* кнопки в тулбарах, теги */
  --h-control-md: 28px; /* стандартная кнопка, input */
  --h-control-lg: 32px; /* primary CTA */

  /* Ширины панелей */
  --sidebar-width-default: 240px;
  --sidebar-width-min: 160px;
  --sidebar-width-max: 480px;
  --tabs-height: 32px;
  --statusbar-height: 22px;
}
```

### Тени

Почти не используем. Тени — признак "материального" дизайна, который в dev-tool выглядит инородно. Используем только для overlay (попапы, dropdown, floating toolbar), и только одну.

```css
:root {
  --shadow-overlay:
    0 4px 16px rgba(0, 0, 0, 0.4), 0 0 0 1px var(--border-default);
}
```

### Анимации

```css
:root {
  --motion-instant: 0ms; /* hover background */
  --motion-fast: 100ms; /* появление tooltip, dropdown */
  --motion-normal: 150ms; /* перерендер блока при F2 */
  --ease-out: cubic-bezier(0.2, 0, 0, 1);
}
```

**Правило:** никаких `transition: all`. Только конкретные свойства (`background`, `border-color`, `opacity`). Никаких анимаций на `height` — только `opacity` и `transform`.

## Компоненты (правила)

### Кнопки

Три варианта: **primary** (accent-фон, используется ровно одна на экран), **secondary** (прозрачный фон, тонкая граница), **ghost** (прозрачный, без границы, только на hover). Все — `--h-control-md` по умолчанию, `--radius-sm`, `--fs-base`, `--fw-medium`.

Floating toolbar использует исключительно **ghost** кнопки размера `--h-control-sm` с иконками 16×16 без текста.

### Inputs

`--h-control-md`, фон `--bg-overlay`, граница `--border-default`, focus — граница `--border-strong` + box-shadow `0 0 0 2px var(--accent-muted)`. Без inset shadow.

### Вкладки

Высота `--tabs-height`. Активная вкладка: фон `--bg-base` (совпадает с областью редактора — создаёт визуальную связь), `--fg-primary`, тонкая полоса `--accent` 2px сверху. Неактивные: фон `--bg-elevated`, `--fg-secondary`. Dirty: `--fg-primary` красного цвета `--danger` + точка-маркер 6px слева от имени. Close-кнопка появляется на hover вкладки.

### Сайдбар (дерево файлов)

Фон `--bg-elevated`, строки высотой 22px, `--fs-sm`, padding `var(--space-1) var(--space-3)`. Hover — `--bg-hover`. Выбранный — `--bg-active` + левая полоса `--accent` 2px. Иконки 14×14 перед именем. Отступ вложенности — `var(--space-3)` на уровень.

### Floating toolbar (форматирование)

Plank с фоном `--bg-overlay`, `--radius-md`, `--shadow-overlay`, padding `var(--space-1)`. Ghost-кнопки `--h-control-sm`, gap `var(--space-1)`. Между группами действий (форматирование / заголовки / код) — разделитель 1px `--border-subtle` с горизонтальным margin `var(--space-1)`.

### Status bar

Высота `--statusbar-height`, фон `--bg-elevated`, `--fs-xs`, `--fg-secondary`. Разделы разделяются вертикальной полосой `--border-subtle`. Интерактивные элементы (toggle переноса строк, режим рендера) — ghost-кнопки без бордера.

### Scroll bars

Используем нативные скроллбары системы, но перекрашиваем в тёмный через `scrollbar-color: var(--border-strong) transparent`. Width 10px, без стрелок.

## Иконки

**Lucide Icons** (https://lucide.dev) — открытая библиотека, 1000+ иконок, есть Svelte-обёртка `lucide-svelte`. Причины: консистентный stroke 2px, нейтральный стиль без "мультяшности", покрывает все нужды dev-tool.

Размер иконок по умолчанию — **16×16**. В floating toolbar — **14×14**. В сайдбаре (иконки файлов) — **14×14**. Цвет наследуется от `currentColor`.

Собственных иконок не рисуем. Если какой-то концепт не покрыт Lucide — ищем ближайший аналог или комбинируем из двух.

## Референсы

Wireframes (если понадобятся) делаем в **Excalidraw** (https://excalidraw.com), сохраняем как PNG и прикрепляем к Linear-задаче. Для простых экранов референс не нужен — достаточно текстового описания расположения блоков в ACL задачи.

Скриншоты существующих приложений, которые нас вдохновляют, собираются в Linear-документ **"mdpad++ Visual References"** — ссылка будет в MDP-34.

## Сценарии использования для примерки

Любой новый компонент проверяется на этих трёх сценариях:

1. **Маленькое окно** (640×400 — минимальный размер). Всё должно оставаться читаемым, сайдбар в collapsed-режиме, тулбар не перекрывает контент.
2. **Большое окно** (1920×1080). Контент не "плавает" посередине бескрайней пустоты — максимальная ширина контента в rendered-режиме ограничена `--prose-max-width: 72ch`.
3. **Очень длинный документ** (5000+ строк). Gutter, диффы, рендер блоков — всё работает без визуальных багов.

## Чек-лист соответствия дизайну (для PR review)

- [ ] Все цвета — через CSS-переменные, нет хардкода `#xxx` в компонентах
- [ ] Все отступы — кратны 4px, через `--space-N`
- [ ] Radius — только из `--radius-*`
- [ ] Шрифт — `--font-ui` или `--font-mono`, не пользовательский
- [ ] Иконки из `lucide-svelte`, размер 14/16
- [ ] Нет анимаций на `all` или `height`
- [ ] Компонент проверен на маленьком окне 640×400
- [ ] Тёмная тема работает (светлая — после MDP-27)

## Что обновлять по мере развития проекта

- При добавлении нового токена — дописывать сюда с объяснением, зачем он нужен.
- При отклонении от правила (например, единственное исключение с бордером 2px) — фиксировать причину.
- Новые компоненты из shadcn-svelte — дописывать в раздел "Компоненты" с правилами их кастомизации под mdpad++.
