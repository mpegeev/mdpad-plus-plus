# Новая задача SENAR

Имя задачи: **$ARGUMENTS**

Я хочу создать новую задачу. Проведи меня через шлюз "Старт" SENAR.

Задай мне следующие вопросы **по одному**, дожидайся ответа на каждый,
прежде чем переходить к следующему. Не пиши код, не редактируй файлы
кроме `tasks/$ARGUMENTS.md`.

## Вопросы

1. **Цель** — какова цель задачи одним предложением? Если ответ длиннее
   одного предложения — попроси сократить.

2. **Критерии приёмки** — перечисли проверяемые условия. Каждый должен
   быть проверяем независимо. Если ответ слишком абстрактный
   ("работает корректно", "хорошо протестирован") — попроси конкретику
   с ожидаемым поведением и значениями.

3. **Негативный сценарий** — какой минимум один негативный сценарий
   (ошибка, некорректный ввод, граничное условие)? Если пользователь
   говорит "нет таких" — предложи сам 2-3 варианта на основе цели и
   попроси выбрать или дополнить.

4. **Область изменений** — какие файлы/модули можно менять? Какие
   НЕЛЬЗЯ трогать? Если ответ слишком общий ("frontend") — попроси
   конкретные пути.

5. **Уровень риска** — задача затрагивает что-либо из:
   - файловые операции (чтение/запись на диск)
   - пользовательские данные (настройки, recovery)
   - внешние API
   - Tauri ACL / capabilities
   - инфраструктуру/CI
   - производительность на больших документах

   На основе ответа определи уровень: standard / high / critical.
   Если есть сомнение — бери уровень выше. Объясни выбор одной фразой.

## После ответов

Создай файл `tasks/$ARGUMENTS.md` со следующей структурой:

```markdown
# $ARGUMENTS

**Статус:** В работе
**Linear:** MDP-XXX (если есть)
**Уровень риска:** <standard|high|critical>
**Создана:** <дата>

## Цель

<одно предложение>

## Критерии приёмки

1. <критерий>
2. <критерий>
   ...

## Негативные сценарии

- <сценарий>

## Область изменений

**Менять:**

- <путь>

**НЕ трогать:**

- <путь или категория>

## Дополнительные ограничения

<если есть: следовать DESIGN.md, не добавлять зависимости, и т.п.>

---

## Свидетельства верификации

<заполняется при закрытии задачи через /done>

## Тупики

<заполняется при закрытии задачи>

## Решения

<заполняется при закрытии задачи>

## Известные проблемы

<заполняется при закрытии задачи>
```

## После создания файла

Покажи супервайзеру содержимое созданного файла и **остановись**.
Не начинай работу над задачей. Дождись отдельной инструкции.

Скажи: "Задача создана в `tasks/$ARGUMENTS.md`. Когда будешь готов
начать работу — дай команду."
