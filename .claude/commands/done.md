# Закрытие задачи через шлюз "Готово"

Имя задачи: **$ARGUMENTS**

Прежде чем отметить задачу завершённой, пройди все шаги шлюза "Готово".
Не пропускай шаги. Не объявляй PASS, пока не пройдены все.

## Шаг 1. Прочитай задачу

Прочитай `tasks/$ARGUMENTS.md`. Если файла нет — поищи Linear issue
с таким именем. Если нет ни файла, ни issue — остановись и сообщи
супервайзеру, что задача не сформулирована (нарушение правила 1).

## Шаг 2. Свидетельства по критериям приёмки

Перечисли каждый критерий приёмки из задачи и для каждого приведи
**конкретное свидетельство**:

- Тестируемые критерии: запусти соответствующий тест через Bash и покажи вывод
- Поведение приложения: запусти `npm run tauri dev` или `npm run dev` и покажи результат
- Изменения в файлах: покажи `git diff` для соответствующего файла
- Линтер: запусти `npm run lint` и `cargo clippy` и покажи вывод
- Дизайн: если UI — проверь соответствие `DESIGN.md`

Если по какому-то критерию свидетельства нет — **скажи это явно** и
останови процесс. Не выдумывай. Не объединяй критерии.

## Шаг 3. Сверка с областью изменений

Запусти `git status` и `git diff --name-only` (если в git) или иначе
получи список всех изменённых файлов. Сравни с разделом "Область
изменений" в задаче.

- Все ли изменённые файлы в разрешённой области?
- Если есть файлы за пределами — **явно перечисли их** и объясни каждый.
- Если ты случайно тронул что-то "по ходу" — это причина для отката.

## Шаг 4. Запуск SENAR-ревьюера

Вызови subagent `senar-reviewer`:

> Use the senar-reviewer subagent to check the changes for task
> $ARGUMENTS at <уровень из задачи> level.

Передай ему:
- описание задачи (из файла или Linear)
- список изменённых файлов
- уровень проверки

Получи структурированный отчёт. **Не интерпретируй его, не сглаживай,
не пересказывай своими словами.** Покажи отчёт супервайзеру дословно.

## Шаг 5. Решение

- Если ревьюер вернул `VERDICT: PASS` и все критерии приёмки имеют
  свидетельства — переходи к шагу 6.
- Если ревьюер вернул `VERDICT: FAIL` — **остановись**. НЕ исправляй
  находки автоматически. Покажи отчёт супервайзеру и дождись инструкций.

## Шаг 6. Фиксация знаний

Заполни в `tasks/$ARGUMENTS.md` разделы:

- **Свидетельства верификации** — копия выводов из шага 2
- **Тупики** — подходы > 15 минут, отвергнутые. Если не было — "нет".
- **Решения** — неочевидные выборы с обоснованием. Если не было — "нет".
- **Известные проблемы** — ограничения, обходные решения. Если нет — "нет".

## Шаг 7. Финализация

Поменяй статус задачи на `Завершена` и поставь дату завершения.

Сообщи супервайзеру:
- Задача `$ARGUMENTS` прошла шлюз "Готово"
- Уровень: `<уровень>`
- Свидетельств собрано: `<число>`
- Зафиксировано тупиков: `<число>`
- Зафиксировано решений: `<число>`

И **остановись**. Не коммить, не пушь — это решает супервайзер.

---

## Что НИКОГДА не делать в этой команде

- Не пропускать шаги "потому что и так понятно"
- Не объявлять PASS, если хоть один критерий без свидетельства
- Не сглаживать отчёт ревьюера — показывать дословно
- Не исправлять находки FAIL автоматически без явной инструкции
